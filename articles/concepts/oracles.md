# <a name="quantum-oracles"></a><span data-ttu-id="80679-103">Oracle Quantum</span><span class="sxs-lookup"><span data-stu-id="80679-103">Quantum Oracles</span></span>

<span data-ttu-id="80679-105">Tali operazioni vengono spesso definite utilizzando una funzione classica $f: \\{0, 1\\} ^ n per \\{0, 1\\} ^ m $, che accetta un $n input binario a $ bit e produce un output binario a $m bit.</span><span class="sxs-lookup"><span data-stu-id="80679-105">Often, such operations are defined using a classical function $f : \\{0, 1\\}^n \to \\{0, 1\\}^m$ which takes an $n$-bit binary input and produces an $m$-bit binary output.</span></span>
<span data-ttu-id="80679-106">A tale scopo, si consideri un particolare input binario $x = (x_{0}, x_{1}, \dots, x_ {n-1}) $.</span><span class="sxs-lookup"><span data-stu-id="80679-106">To do so, consider a particular binary input $x = (x_{0}, x_{1}, \dots, x_{n-1})$.</span></span>
<span data-ttu-id="80679-107">È possibile etichettare gli Stati qubit come $ \ket{\vec{x}} = \ket{x_{0}} \otimes \ket{x_{1}} \otimes \cdots \otimes \ket{x_ {n-1}} $.</span><span class="sxs-lookup"><span data-stu-id="80679-107">We can label qubit states as $\ket{\vec{x}} = \ket{x_{0}} \otimes \ket{x_{1}} \otimes \cdots \otimes \ket{x_{n-1}}$.</span></span>

<span data-ttu-id="80679-108">È possibile che si tenti prima di tutto di definire $O $ in modo che $O \ket{x} = \ket{f (x)} $, ma ciò presenta un paio di problemi.</span><span class="sxs-lookup"><span data-stu-id="80679-108">We may first attempt to define $O$ so that $O\ket{x} = \ket{f(x)}$, but this has a couple problems.</span></span>

<span data-ttu-id="80679-110">In secondo luogo, anche se $n = m $, la funzione potrebbe non essere invertibile: se $f (x) = f (y) $ per alcuni $x \ne y $, quindi $O \ket{x} = O\ket {y} $ ma $O ^ \dagger O\ket {x} \ne O ^ \dagger O\ket {y} $.</span><span class="sxs-lookup"><span data-stu-id="80679-110">Second, even if $n = m$, the function may not be invertible: if $f(x) = f(y)$ for some $x \ne y$, then $O\ket{x} = O\ket{y}$ but $O^\dagger O\ket{x} \ne O^\dagger O\ket{y}$.</span></span>

## <a name="defining-an-oracle-by-its-effect-on-computational-basis-states"></a><span data-ttu-id="80679-112">Definizione di un Oracle per effetto sugli stati di base di calcolo</span><span class="sxs-lookup"><span data-stu-id="80679-112">Defining an oracle by its effect on computational basis states</span></span>

<span data-ttu-id="80679-114">Si definirà quindi l'effetto di Oracle su tutti gli Stati di base computazionale: per tutti $x \In \\{0, 1\\} ^ n $ e $y \In \\{0, 1\\} ^ m $,</span><span class="sxs-lookup"><span data-stu-id="80679-114">Then we will define the effect of the oracle on all computational basis states: for all $x \in \\{0, 1\\}^n$ and $y \in \\{0, 1\\}^m$,</span></span>

<span data-ttu-id="80679-115">$ $ \begin{align} O (\ket{x} \otimes \ket{y}) = \ket{x} \otimes \ket{y \oplus f (x)}.</span><span class="sxs-lookup"><span data-stu-id="80679-115">$$ \begin{align} O(\ket{x} \otimes \ket{y}) = \ket{x} \otimes \ket{y \oplus f(x)}.</span></span>
<span data-ttu-id="80679-116">\end{align} $ $</span><span class="sxs-lookup"><span data-stu-id="80679-116">\end{align} $$</span></span>

> [!TIP]
> <span data-ttu-id="80679-118">Per verificare che $O = O ^ {\dagger} $, si noti che $O ^ 2 = \boldone $ da $a \oplus b \oplus b = a $ per tutti $a, b \In \{0, 1\}$.</span><span class="sxs-lookup"><span data-stu-id="80679-118">To see that $O = O^{\dagger}$, note that $O^2 = \boldone$ since $a \oplus b \oplus b = a$ for all $a, b \in \{0, 1\}$.</span></span>
> <span data-ttu-id="80679-119">Di conseguenza, $O \ket{x} \ket{y \oplus f (x)} = \ket{x} \ket{y \oplus f (x) \oplus f (x)} = \ket{x} \ket{y} $.</span><span class="sxs-lookup"><span data-stu-id="80679-119">As a result, $O \ket{x} \ket{y \oplus f(x)} = \ket{x} \ket{y \oplus f(x) \oplus f(x)} = \ket{x} \ket{y}$.</span></span>

<span data-ttu-id="80679-120">In particolare, la definizione di un Oracle in questo modo per ogni stato di base computazionale $ \ket{x}\ket{y} $ definisce anche il modo in cui $O $ agisce per qualsiasi altro stato.</span><span class="sxs-lookup"><span data-stu-id="80679-120">Importantly, defining an oracle this way for each computational basis state $\ket{x}\ket{y}$ also defines how $O$ acts for any other state.</span></span>

## <a name="phase-oracles"></a><span data-ttu-id="80679-132">Oracle della fase</span><span class="sxs-lookup"><span data-stu-id="80679-132">Phase oracles</span></span>

<span data-ttu-id="80679-139">\end{align} $ $</span><span class="sxs-lookup"><span data-stu-id="80679-139">\end{align} $$</span></span>
