# <a name="licensing"></a>Licenze #

Quantum Development Kit viene fornito con un'ampia raccolta di funzioni e operazioni open source, concessa in licenza con la [licenza mit](https://github.com/Microsoft/Quantum/blob/master/LICENSE.txt).
Le parti della libreria standard portabili tra i computer di destinazione possono essere ottenute dal repository [Microsoft/QuantumLibraries](https://github.com/Microsoft/QuantumLibraries) su GitHub, insieme ad altre librerie come e ad altre librerie, ad esempio la [libreria Quantum Chemistry ](xref:microsoft.quantum.chemistry.concepts.intro).

Microsoft Quantum Development Kit fornisce anche funzioni e operazioni di libreria specializzate che vengono concesse in licenza con una [licenza di Microsoft Research](https://github.com/Microsoft/Quantum-NC/blob/master/LICENSE).
Questi possono essere ottenuti dal repository [Microsoft/Quantum-NC](https://github.com/microsoft/quantum-nc) su GitHub.

Sono inoltre disponibili molti esempi che illustrano l'utilizzo di funzioni e operazioni dalla libreria standard e da altre librerie.
Questi esempi sono concessi in licenza con la [licenza mit](https://github.com/Microsoft/Quantum/blob/master/LICENSE.txt).
Gli esempi possono essere ottenuti dal repository [Microsoft/Quantum](https://github.com/Microsoft/Quantum) su GitHub.

## <a name="contributing"></a>Contributo ##

Questo progetto accoglie i contributi e i suggerimenti.
Per la maggior parte dei contributi è necessario accettare un contratto di licenza con collaboratore che dichiara di avere il diritto di concedere a Microsoft i diritti per l'utilizzo del contributo. Per informazioni dettagliate, vedere [CLA](https://cla.microsoft.com).

Quando si invia una richiesta pull, un CIA-bot determinerà automaticamente se è necessario fornire un CLA e decorare la richiesta pull in modo appropriato (ad esempio, etichetta, commento). È sufficiente seguire le istruzioni fornite dal bot. Questa operazione dovrà essere eseguita solo una volta per tutti i repository tramite il nostro CLA.

Questo progetto ha adottato il [Microsoft Open Source Code of Conduct](https://opensource.microsoft.com/codeofconduct/) (Codice di comportamento Open Source di Microsoft).
Per altre informazioni, vedere [Code of Conduct FAQ](https://opensource.microsoft.com/codeofconduct/faq/) (Domande frequenti sul Codice di comportamento Open Source di Microsoft) oppure contattare [opencode@microsoft.com](mailto:opencode@microsoft.com) per eventuali altre domande o commenti.
