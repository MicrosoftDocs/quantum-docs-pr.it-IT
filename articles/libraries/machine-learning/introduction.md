### <a name="likelihood-as-the-training-goal"></a><span data-ttu-id="a9b79-141">Probabilità dell'obiettivo di training</span><span class="sxs-lookup"><span data-stu-id="a9b79-141">Likelihood as the training goal</span></span>

<span data-ttu-id="a9b79-142">Considerato un circuito Quantum $U (\theta) $, dove $ \theta $ è un vettore di parametri e indicante la misurazione finale per $M $, la probabilità media dell'inferenza dell'etichetta corretta è $ $ \begin{align} \mathcal{L} (\theta) = \frac{1}{| \mathcal{D} |} \left (\ sum_ {(x, y_1) \in\mathcal{D}} P (M = y_1 | U (\theta) x) + \ sum_ {(x, y_2) \in\mathcal{D}} P (M = y_2 | U (\theta) x) \right) \end{align} $ $ where $P (M = y | z) $ è la probabilità di misurare $y $ nello stato quantum $z $.</span><span class="sxs-lookup"><span data-stu-id="a9b79-142">Given a learnable quantum circuit $U(\theta)$, where $\theta$ is a vector of parameters, and denoting the final measurement by $M$, the average likelihood of the correct label inference is $$ \begin{align} \mathcal{L}(\theta)=\frac{1}{|\mathcal{D}|} \left( \sum_{(x,y_1)\in\mathcal{D}} P(M=y_1|U(\theta) x) + \sum_{(x,y_2)\in\mathcal{D}} P(M=y_2|U(\theta) x)\right) \end{align} $$ where $P(M=y|z)$ is the probability of measuring $y$ in quantum state $z$.</span></span>
<span data-ttu-id="a9b79-143">In questo caso è sufficiente comprendere che la funzione di probabilità $ \mathcal{L} (\theta) $ è smussata in $ \theta $ e il relativo derivato in qualsiasi $ \ theta_j $ può essere calcolato essenzialmente dallo stesso protocollo Quantum usato per calcolare la funzione di probabilità stessa.</span><span class="sxs-lookup"><span data-stu-id="a9b79-143">Here, it suffices to understand that the likelihood function $\mathcal{L}(\theta)$ is smooth in $\theta$ and its derivative in any $\theta_j$ can be computed by essentially the same quantum protocol as used for computing the likelihood function itself.</span></span> <span data-ttu-id="a9b79-144">In questo modo è possibile ottimizzare $ \mathcal{L} (\theta) $ per Descent gradient.</span><span class="sxs-lookup"><span data-stu-id="a9b79-144">This allows for optimizing the $\mathcal{L}(\theta)$ by gradient descent.</span></span>

### <a name="classifier-bias-and-training-score"></a><span data-ttu-id="a9b79-145">Distorsione del classificatore e Punteggio di training</span><span class="sxs-lookup"><span data-stu-id="a9b79-145">Classifier bias and training score</span></span>

<span data-ttu-id="a9b79-146">Dato alcuni valori intermedi (o finali) dei parametri in $ \theta $, è necessario identificare un singolo valore reale $b $ know come bias di *classificazione* per eseguire l'inferenza.</span><span class="sxs-lookup"><span data-stu-id="a9b79-146">Given some intermediate (or final) values of the parameters in $\theta$, we need to identify a single real value $b$ know as *classifier bias* to do the inference.</span></span> <span data-ttu-id="a9b79-147">La regola di inferenza delle etichette funziona nel modo seguente:</span><span class="sxs-lookup"><span data-stu-id="a9b79-147">The label inference rule works as follows:</span></span> 
- <span data-ttu-id="a9b79-148">A un esempio $x $ viene assegnata un'etichetta $y _2 $ if e solo se $P (M = y_2 | U (\theta) x) + b > $0,5 (RULE1). in caso contrario, viene assegnata l'etichetta $y _1 $)</span><span class="sxs-lookup"><span data-stu-id="a9b79-148">A sample $x$ is assigned label $y_2$ if and only if $P(M=y_2|U(\theta) x) + b > 0.5$  (RULE1) (otherwise it is assigned label $y_1$)</span></span>

<span data-ttu-id="a9b79-149">Ovviamente $b $ deve essere compreso nell'intervallo $ (-0,5, + 0,5) $ per essere significativo.</span><span class="sxs-lookup"><span data-stu-id="a9b79-149">Clearly $b$ must be in the interval $(-0.5,+0.5)$ to be meaningful.</span></span>

<span data-ttu-id="a9b79-150">Un case di training $ (x, y) \in \mathcal{D} $ è considerato una *classificazione errata* in base alla distorsione $b $ se l'etichetta dedotta per $x $ come per Rule1 è effettivamente diversa da $y $.</span><span class="sxs-lookup"><span data-stu-id="a9b79-150">A training case $(x,y) \in \mathcal{D}$ is considered a *misclassification* given the bias $b$ if the label inferred for $x$ as per RULE1 is actually different from $y$.</span></span> <span data-ttu-id="a9b79-151">Il numero complessivo di classificazioni non configurate è il *Punteggio di training* del classificatore data la distorsione $b $.</span><span class="sxs-lookup"><span data-stu-id="a9b79-151">The overall number of misclassifications is the *training score* of the classifier given the bias $b$.</span></span> <span data-ttu-id="a9b79-152">La distorsione di classificazione *ottimale* $b $ riduce al minimo il Punteggio di training.</span><span class="sxs-lookup"><span data-stu-id="a9b79-152">The *optimal* classifier bias $b$ minimizes the training score.</span></span> <span data-ttu-id="a9b79-153">È facile vedere che, date le stime di probabilità pre-calcolate $\{ P (M = y_2 | U (\theta) x) | (x, \*) \in\mathcal{D} \}$, la distorsione ottimale del classificatore è disponibile nella ricerca binaria nell'intervallo $ (-0.5, + 0,5) $ facendo al massimo $ \ log_2 (| \mathcal{D} |) $ Steps.</span><span class="sxs-lookup"><span data-stu-id="a9b79-153">It is easy to see that, given the precomputed probability estimates $\{ P(M=y_2|U(\theta) x) | (x,\*)\in\mathcal{D} \}$, the optimal classifier bias can be found by binary search in interval $(-0.5,+0.5)$ by making at most $\log_2(|\mathcal{D}|)$ steps.</span></span>
