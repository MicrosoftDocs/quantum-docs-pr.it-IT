# <a name="reporting-bugs"></a>Segnalazione di bug #

Se si ritiene di aver trovato un bug in un componente del quantum Development Kit, è molto apprezzato un report.
Dopotutto, qualcun altro potrebbe avere difficoltà anche nello stesso problema; ci consentirà di aiutarci a risolverlo per tutti.
Il primo passaggio nella creazione di report di un bug è iniziare esaminando i problemi esistenti rilevati in ogni repository, perché è molto probabile che qualcun altro abbia segnalato lo stesso problema.

Se il problema è già stato rilevato in precedenza, potrebbe essere utile fornire informazioni aggiuntive o un contesto che consenta di diagnosticare e risolvere il problema.
In tal caso, è possibile partecipare alle discussioni sul problema.

Se il bug non è stato segnalato in precedenza, si apprezzerebbe se si apre un nuovo problema nel repository per il componente in questione.
Quando si crea un nuovo problema, viene visualizzato un modello che suggerisce alcune informazioni spesso utili per la creazione di report sui bug informativi:

- Passi per riprodurre il problema
- Cosa dovrebbe accadere
- Cosa è effettivamente successo
- Informazioni sull'ambiente software (ad esempio, versioni del sistema operativo, .NET Core e Quantum Development Kit)

È anche possibile [creare una richiesta pull](https://help.github.com/articles/about-pull-requests/) per correggere direttamente il bug, se è molto semplice e non vale la pena discutere (ad esempio, un errore di digitazione).
